Python¶

If you are using the university cloud python, navigate to the class canvas homepage and click on the Online JupyterHub link. This will launch your own individual server. Poke around, make sure you can open a terminal to the (virtual) Linux machine, then open a JupyterHub session and skip down to New notebook below.

If you are running python on your own machine, make sure you have a good environment. There are various ways of doing this, but I'd suggest using conda (command line) or Anaconda (GUI wrapper). Conda is an environment manager. That means you can make different 'installations' of python and all the various libraries, each isolated into its own environment. This is really nice for code hygene, as incompatible sets of libraries can be isolated from each other.

If you use the command line, defining a .yaml file can be really nice, then you can create the environment (and blow it away and recreate it) in one well documented step. Here is a copy of my .yaml file

name: scratch
channels:
  - defaults
  - conda-forge
dependencies:
  - astropy
  - astropy-healpix
  - ipython
  - jupyter
  - jupyterlab
  - matplotlib
  - scipy
  - numpy
  - pandas
  - pyuvdata
  - pyyaml
  - yaml

Once you have an environment, make sure it is the one you are using (activated), then launch your jupyter session.

JupyterLab or jupyter notebook¶

JupyterLab and jupyter notebooks are very closely relatedthey are really just different interfaces to the same underlying notebooks. Pick whichever you like (the online defaults to JupyterLab). If that opens a new window in your web browser, you are in luck and go to the next step. If not, more online trouble shooting will be needed.

New notebook¶

In either environment, you see the list of notebooks. To actually start a notebook find the New command (button upper right in notebooks; File menu->New for JupyterLab), and select python 3. This should open a new Notebook. Make sure to save it with a name. You'll want a new notebook for every homework assignment and Lab.

Simple statistics¶

Okay, now for some simple statistics to make sure you're computing environment is setup and to practice a few commands we'll use over and over again.

First, let's draw a 100k points from a Gaussian distribution. In many areas of statistics a Gaussian distribution is called the 'Normal distribution'. So we'll dig into the stats package from scipy (see import lines above). Read up on the command to make sure you understand what it is doing.

In [2]:
d = stats.norm.rvs(loc = 5., scale = 0.01, size = 100000)

Now let's plot a histogram of these points. Make sure you understand what each of these commands is doing.

In [3]:
fig, ax = plt.subplots(1, 1)
ax.hist(d,50, density=True)
plt.tick_params(labelsize = 24)
plt.xlim([4.95,5.05])
x = np.linspace(4.95,5.05,1000)
ax.plot(x,stats.norm.pdf(x,loc = 5., scale = 0.01),linewidth = 8,alpha = 0.7)
plt.show()
